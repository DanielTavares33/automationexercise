# Automation Exercise

E2E test automation for [automationexercise.com](https://automationexercise.com) using Playwright BDD.

## Stack

- [Playwright](https://playwright.dev/) — browser automation and test runner
- [playwright-bdd](https://github.com/vitalets/playwright-bdd) — Gherkin BDD on top of `@playwright/test`
- TypeScript

## Project Structure

```
features/           # Gherkin .feature files (organized by domain)
locators/           # Locator classes per page
pages/              # Page Object Models (extend BasePage)
step_definitions/   # Step implementations using playwright-bdd
support/
  fixtures.ts       # Custom Playwright fixtures (homePage, loginPage, registerPage)
.features-gen/      # Auto-generated by playwright-bdd (gitignored)
```

## Setup

```bash
npm install
npx playwright install chromium
```

## Running Tests

### CI/Parallel Execution (Recommended)

```bash
# Run all tests in parallel, headless mode, with 2 retries
./scripts/run-tests.sh

# Or via npm
npm run test:ci

# Run specific test group
./scripts/run-tests.sh smoke      # @smoke scenarios
./scripts/run-tests.sh regression # full suite
./scripts/run-tests.sh login      # login feature only

# Custom worker count
WORKERS=8 ./scripts/run-tests.sh

# Combine custom workers with tag filter
WORKERS=8 ./scripts/run-tests.sh smoke
```

### Interactive Development

```bash
# Run all tests
npm test

# Run by tag group
npm run test:smoke       # @smoke scenarios — fast CI gate
npm run test:regression  # full suite
npm run test:login       # login feature only
npm run test:register    # register feature only

# Run in parallel (N workers)
WORKERS=4 npm test

# Run headless
HEADLESS=true npm test

# Combine
WORKERS=4 HEADLESS=true npm run test:smoke
```

## Tag Strategy

| Dimension | Tags | Usage |
|---|---|---|
| Execution tier | `@smoke`, `@regression` | CI filtering |
| Domain | `@login`, `@register` | Feature area |
| Outcome | `@positive`, `@negative` | Test type |

## Test Cases

| Scenario | Tags |
|---|---|
| Register user (TC1) | `@smoke @positive @register` |
| Login with correct credentials (TC2) | `@smoke @positive @login` |
| Login with incorrect credentials (TC3) | `@regression @negative @login` |
| Logout user (TC4) | `@smoke @positive @login` |
| Register with existing email (TC5) | `@regression @negative @register` |

## Writing Tests

1. Add a `.feature` file in `features/`
2. Add a locator class in `locators/` with `readonly Locator` fields (CSS selectors only)
3. Add a page object in `pages/` extending `BasePage`
4. Register the page object as a fixture in `support/fixtures.ts`
5. Add step definitions in `step_definitions/` using `createBdd(test)`
